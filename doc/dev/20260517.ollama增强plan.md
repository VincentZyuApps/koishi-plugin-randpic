# Randpic 插件增强计划：Ollama 视觉模型集成

## 一、现状分析

### 当前架构
```
索引流程（现在）：
扫描图片 → 提取文件名 → 本地 BERT 模型转成向量 → 存入 Qdrant

搜索流程（现在）：
用户输入关键词 → 本地 BERT 模型转成向量 → Qdrant 查相似向量 → 返回结果
```

### 核心问题
- 向量是基于**文件名文本**生成的，不是基于图片内容
- 如果文件名是 `IMG_20231015_001.jpg` 或与实际内容无关，向量搜索效果极差
- 本质上是"按文件名语义搜索"，而非"按图片内容搜索"

---

## 二、增强方案

### 核心思路
**让索引阶段"看懂"图片内容，生成基于视觉语义的向量**

```
索引流程（增强后）：
扫描图片 → 读取图片 base64 → Ollama 视觉模型分析 → 生成描述性 tag
→ tag + 文件名组合 → 本地 BERT 模型转成向量 → 存入 Qdrant

搜索流程（不变，依然快）：
用户输入关键词 → 本地 BERT 模型转成向量 → Qdrant 查相似向量 → 返回结果
```

### 为什么搜索不需要 GPU？
- **GPU 加速在 Ollama 那边**：索引时调用 Ollama 视觉模型，如果你的 Ollama 配置了 GPU，自动使用
- **搜索阶段不需要 GPU**：Qdrant 查向量已经是计算好的，毫秒级响应
- **本地嵌入模型**：384 维的 BERT 模型在 CPU 上跑也很快（单次推理 < 100ms）

### 2. src/ollama.ts - 新增 Ollama 服务类

**功能：**
- 调用 Ollama `/api/generate` 接口（支持 vision 模型）
- 传入图片 base64 + prompt → 返回描述性 tag
- 支持批量处理和进度回调
- 错误处理和重试机制

**核心方法：**
```typescript
export class OllamaVisionService {
  private ctx: Context
  private config: Config
  private baseUrl: string

  constructor(ctx: Context, config: Config) {
    this.ctx = ctx
    this.config = config
    this.baseUrl = `http://${config.ollamaHost}:${config.ollamaPort}`
  }

  /**
   * 分析单张图片，返回描述性 tag
   * @param imageBase64 图片 base64 数据
   * @returns 描述文本（如 "橘猫 可爱 草地 晒太阳 户外"）
   */
  async analyzeImage(imageBase64: string): Promise<string>

  /**
   * 批量分析图片
   * @param images 图片 base64 数组
   * @param onProgress 进度回调
   * @returns 描述文本数组
   */
  async analyzeBatch(
    images: string[], 
    onProgress?: (current: number, total: number) => void
  ): Promise<string[]>

  /**
   * 检查 Ollama 服务是否可用
   */
  async isAvailable(): Promise<boolean>
}
```

**API 调用示例（Ollama /api/generate）：**
```json
POST http://127.0.0.1:11434/api/generate
{
  "model": "llava:7b",
  "prompt": "请详细描述这张图片的内容...",
  "images": ["base64encodedimage..."],
  "stream": false
}
```

### 3. src/search.ts - 修改索引流程

**修改 `indexImages()` 方法：**

```typescript
async indexImages(onProgress?: (message: string) => void): Promise<number> {
  // ... 现有检查逻辑 ...

  // 重新扫描图片
  const images = this.scanImages()
  
  // 决定是否使用 Ollama 视觉分析
  const useVision = this.config.enableOllamaVision
  
  if (useVision) {
    onProgress?.(`找到 ${images.length} 张图片，开始 Ollama 视觉分析...`)
    
    // 1. 批量调用 Ollama 生成描述
    const imageBase64s = images.map(img => this.getImageBase64(img.filepath))
    const descriptions = await this.ollama.analyzeBatch(imageBase64s, (current, total) => {
      onProgress?.(`视觉分析中: ${current}/${total}`)
    })
    
    // 2. 组合文本：描述 + 文件名
    const texts = images.map((img, i) => {
      const name = img.virtualName
        .replace(/\.[^.]+$/, '')
        .replace(/[_\-\/]/g, ' ')
      return `${descriptions[i]} ${name}`  // 描述在前，文件名在后
    })
  } else {
    // 原有逻辑：仅用文件名
    const texts = images.map(img => { ... })
  }
  
  // 3. 生成向量（不变）
  const vectors = await this.embedding.embedBatch(texts, ...)
  
  // 4. 存入 Qdrant（payload 增加 tags 字段）
  const points = images.map((img, i) => ({
    id: img.filepath,
    filename: img.filename,
    filepath: img.filepath,
    vector: vectors[i],
    tags: useVision ? descriptions[i] : null,  // 新增：存储生成的 tag
  }))
  
  // ... 上传 Qdrant（不变）...
}
```

## 五、依赖和前置条件

### 用户需要准备的
1. **Ollama 已安装并运行**：`http://127.0.0.1:11434`
2. **已下载视觉模型**：如 `ollama pull llava:7b` 或 `ollama pull bakllava`
3. **GPU 推荐但非必须**：Ollama 会自动使用 GPU（如果有），CPU 也能跑但较慢

### 插件新增依赖
- **无新增 npm 依赖**：使用 Node.js 原生 `fetch` 或 Koishi 的 `ctx.http` 调用 Ollama API

## 八、注意事项

### 性能考虑
- **索引速度**：启用 Ollama 后，索引速度取决于 Ollama 推理速度
  - GPU：约 1-3 秒/张图片
  - CPU：约 5-15 秒/张图片
- **搜索速度**：不变（毫秒级，Qdrant 直接查向量）
- **建议**：索引完成后可以关闭 Ollama，搜索不需要它

### 错误处理
- Ollama 服务不可用时，自动降级为仅使用文件名
- 单张图片分析失败时，记录日志并跳过，不影响整体索引
- 支持超时配置，避免长时间卡住

### 兼容性
- 默认 `enableOllamaVision: false`，不影响现有用户
- 现有配置和索引数据完全兼容
- 可以随时开启/关闭 Ollama，重新索引即可
